UIC APP v0.6 (FULL)

Qué se corrigió / mejoró:
1) WordPress API (error 401 "itsec_rest_api_access_restricted")
   - El backend ahora soporta Basic Auth opcional para consumir WP REST API.
   - Nuevo endpoint de diagnóstico: GET /wp/ping (devuelve el JSON crudo de WP, sirve para confirmar si WP responde o sigue bloqueando).

2) Push (Web Push)
   - Se DESACTIVA en el MVP para evitar confusión durante el onboarding.
   - Se elimina el botón "Activar Push" y no se intenta registrar push en el frontend.
   - El backend conserva su lógica de push, pero si VAPID está mal, queda deshabilitado sin romper el servicio.

Variables de entorno (Render)
A) Backend (Web Service) uic-campana-api
   ALLOWED_ORIGINS=https://uic-campana-app.onrender.com
   WP_BASE=https://uic-campana.com.ar/wp-json/wp/v2

   Si WP está bloqueado por iThemes Security, usar UNA opción:
   Opción 1) WP_USER + WP_APP_PASSWORD
     WP_USER=tu_usuario_wp
     WP_APP_PASSWORD=xxxx xxxx xxxx xxxx xxxx xxxx

   Opción 2) WP_AUTH_B64
     WP_AUTH_B64=base64("user:app_password")

B) Frontend (Static Site) uic-campana-app
   VITE_API_BASE=https://uic-campana-api.onrender.com

Cómo probar rápido
1) Probar WP directo (en navegador):
   https://uic-campana.com.ar/wp-json/wp/v2/posts?per_page=1
   Si devuelve 401 itsec_..., está bloqueado.

2) Probar el proxy del backend:
   https://uic-campana-api.onrender.com/wp/ping
   - Si sin credenciales devuelve 401 => falta permitir REST en WP o configurar auth.
   - Si con credenciales devuelve array de posts => OK.
